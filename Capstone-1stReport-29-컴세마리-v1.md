# Project-Summary
| 항목 | 내용 |
|:---  |---  |
| (1) 문제 정의 |  1. 여가 정보들이 흩어져 있다 <br> 2. 여가 정보를 찾을 시간이 없다 <br> 3. 내 일정에 맞는 여가를 찾기가 어렵다 <br> 4. 여가 선택지가 많아서 결정하기 어렵다 <br> 5. 자신의 니즈에 맞는 여가를 모른다 <br> |
| (2) 기존연구와의 비교 | *유사한 과제/연구/서비스/시스템의 예를 들고, 각각의 장단점을 기술할 것. 특히, 본 과제가 유사과제에 대하여 갖는 장점을 부각할 것* |
| (3) 제안 내용 | 1. 한 번에 모아볼 수 있는 통합 플랫폼 <br> 2. 내 일정에 맞는 여가를 검색할 수 있는 서비스 <br> 3. 내 취향에 맞는 여가를 추천해주는 서비스 <br> 4. 내가 예약할만한 여가를 빠르게 추천해주는 서비스 |

| (5) 주요 기능 리스트 | **1.영화/공연/전시 API를 연동해 통합** : 흩어져 있는 여가 데이터를 한 곳에서 모아 볼 수 있도록 한다. <br><br> **2.여가 통합 검색** : 영화/공연/전시 API에서 받아온 데이터 중 사용자가 검색한 키워드에 해당하는 여가 리스트를 반환한다. <br><br> **3.추천 시스템** : 협력 필터링과 콘텐츠 기반 추천을 함께 사용하는 하이브리드 추천 알고리즘을 사용한다. 사용자의 취향 데이터를 활용해 사용자 맞춤형 추천 여가 리스트를 작성한다. <br><br> **4.맞춤형 여가 추천 리스트** : 추천 알고리즘에 의해 선정된 추천 리스트를 반환한다. 카테고리별, 거리순, 리뷰순 정렬을 제공한다. <br><br> **5.사용자 취향 데이터 수집용 챗봇** : 사용자와 챗봇 간의 대화를 통해 취향 데이터를 키워드 형식으로 추출한다. 챗봇은 사용자가 이전에 경험했던 여가나 좋아하는 장르와 같은 대화 주제를 활용하여 사용자의 선호 정보를 직·간접적으로 이끌어낸다. <br><br> **6.사용자 취향 프로필** : 챗봇과의 대화에서 추출한 키워드와 사용자가 이전에 남긴 평점 및 리뷰를 통해 사용자의 취향 프로필을 작성해 저장·업데이트하고 추천 알고리즘에 반영한다. <br><br> **7.평점 및 리뷰작성** : 사용자가 경험한 여가들에 대해 자신의 만족도를 평점으로 남기고, 리뷰를 작성할 수 있다. 추천 알고리즘에 반영되며, 여가 탐색 시 여가 정보와 함께 사용자들이 남긴 리뷰를 볼 수 있다. <br><br> **8.여가 찜하기** : 마음에 드는 여가를 찜하고 한곳에서 모아볼 수 있다. <br><br> **9.여가 예약등록** : 예약한 여가를 한곳에서 모아볼 수 있도록 날짜를 등록한다. <br><br> **10.소셜 로그인** : 사용자의 취향 프로필을 생성하고, 정보를 효율적으로 저장 및 불러오기 위해 사용한다. <br><br> **11.구글 캘린더 연동** : 사용자의 일정을 불러와 일정과 겹치는 시간의 여가를 제외하고 볼 수 있게 한다. <br><br> **12.지도 API 연동** : 리스트에 포함된 여가 시설을 사용자의 위치와의 거리 순으로 정렬하기 위해 사용한다. |

# Project-Design & Implementation
| 항목 | 내용 |

| (2) 전체 시스템 구성 | *프로젝트를 위하여, SW 전체 시스템의 구조를 보인다. (가능하다면, 사용자도 포함) <br> 주요 SW 모듈을 보이고, 각각의 역할을 기술한다. <br>만약, 오픈소스 혹은 외부 모듈을 사용한다면 이또한 기술한다.* |
| (3) 주요엔진 및 기능 설계 | *프로젝트의 주요 기능 혹은 모듈의 설계내용에 대하여 기술한다 <br> SW 구조 그림에 있는 각 Module의 상세 구현내용을 자세히 기술한다.* |
| (4) 주요 기능의 구현 | **1.추천 시스템** <br><br> **1-1.사용 라이브러리** <br> - **scikit-learn** : `cosine_similarity` 메서드를 활용하여 키워드 형식의 사용자 취향 데이터와 아이템(여가) 간의 유사도를 계산한다. <br> - **서프라이즈 (Surprise)** : 사용자-아이템 평점 데이터를 분석한 뒤, SVD(행렬 분해) 기법을 통해 각 사용자와 아이템의 잠재적 요인을 추출한다. 이를 기반으로 사용자가 평가하지 않은 아이템의 평점을 예측한다. <br><br> **1-2.구현 과정** <br> &nbsp;&nbsp; **a. 초기 단계 - 데이터가 부족해 협력 필터링 추천이 어려운 경우** <br> &nbsp;&nbsp;&nbsp;&nbsp;1.**사용자 취향 데이터 수집**- 챗봇과 사용자의 대화를 통해 사용자의 취향 데이터(키워드)를 수집해 사용자 취향 프로필을 작성한다. <br> &nbsp;&nbsp;&nbsp;&nbsp;2.**유사도 계산**- 각 아이템(영화/공연/전시)들과 사용자 취향 데이터 간의 유사도를 계산한다. <br> &nbsp;&nbsp;&nbsp;&nbsp;3.**추천 리스트 생성**- 높은 유사도를 보이는 아이템들로 추천 리스트를 만들어 출력한다. 유사도 계산 결과가 대체로 낮게 나와 양적으로 충분한 추천 리스트를 생성하지 못하면 일반적으로 높은 선호도를 보이는(인기 있는) 여가를 함께 추천한다. <br><br> &nbsp;&nbsp; **b. 평점 데이터가 충분히 모여 협력 필터링 추천이 가능한 경우** <br> &nbsp;&nbsp;&nbsp;&nbsp;1.**SVD 모델 학습 및 예측**- 평점 데이터를 기반으로 서프라이즈 패키지 내의 SVD 모델을 학습한다. 학습된 모델을 이용해 각 아이템 별 사용자의 예측 평점을 계산한다. <br> &nbsp;&nbsp;&nbsp;&nbsp;2.**추천 후보군 추출**- 예측 평점은 사용자가 관심을 가질 가능성이 높은 아이템을 찾는 데 사용되며, 이를 바탕으로 상위 아이템을 추천 후보군으로 추출한다. <br> &nbsp;&nbsp;&nbsp;&nbsp;3.**유사도 기반 최종 추천**- 이전 단계(a-2)에서 계산된 코사인 유사도를 활용해 추천 후보군 아이템 중 높은 유사도의 아이템을 추천 리스트에 넣어 출력한다. 유사도를 모두 고려한 최종 추천 리스트를 생성해 출력한다. <br><br><br> **2.사용자 취향 데이터 수집용 챗봇** <br><br> **1-1.사용 API** : Open AI API (GPT 3.5-turbo) <br><br> **1-2.대화 흐름 설계** : 사용자의 취향과 선호 정보를 수집할 수 있도록 적절한 질문을 제시한다. 초기 단계에서는 일반적이고 포괄적인 질문을 통해 사용자의 관심사를 탐색하며, 이후 사용자의 응답 내용을 기반으로 구체화된 추가 질문을 생성한다. <br><br> **1-3.자연어 처리를 활용한 감성 분석 및 긍정 키워드 추출** : 감성 분석 기술을 활용하여 사용자의 응답에서 긍정적인 반응을 나타내는 키워드를 추출하고, 사용자 취향 프로필에 저장한다. <br>|
